# Community And Go-To-Market Notes

These notes are for public positioning and early community learning. They are not a pricing plan or a commitment to launch a paid product.

## Initial Audience

Start with users who already feel workflow pain:

- Codex power users
- solo founders
- small dev teams
- AI-assisted agencies
- people building websites from reference sites
- teams worried about external AI disclosure

## Launch Channels

- GitHub README and examples
- X/Twitter build-in-public posts
- Hacker News "Show HN" once examples are polished
- Reddit communities for Codex/AI coding only if posts are practical, not spammy
- YouTube/short video demos
- Blog posts comparing raw prompts vs routed workflows

## First Five Demos

1. Reference website to multi-AI council to differentiated spec.
2. Screenshot bug to root cause to verified fix.
3. GitHub CI failure triage.
4. Context overflow to handoff recovery.
5. Creative asset request with missing details; clarification before generation.

## Metrics To Track Manually First

- Time to first useful route.
- Number of clarifying questions avoided.
- Token/context saved by progressive disclosure.
- Tasks closed with verification evidence.
- Number of user-contributed capability cards.
- Repeated workflows worth turning into maintained packs.

## Early Sustainability Signals

Strong signals:

- Teams ask for private profiles.
- Users ask for policy/allowlist controls.
- Users want maintained packs for their stack.
- Agencies ask for repeatable website build workflows.
- Users want route traces exported for clients or managers.

Weak signals:

- People only star the repo but do not install.
- Users ask for "more prompts" but not workflow integration.
- Requests require many one-off customizations that do not generalize.

## Possible First Service Offer

If teams ask for hands-on help, start with service-led validation:

```text
Private gstack setup for your team: capability cards, project profile, verification standards, and workflow handoff docs.
```

Then turn repeated setup work into maintained packs only after real demand is clear.

## Community Strategy

Use `docs/community-strategy.md` as the channel plan. GitHub remains the source of truth; community posts should drive users back to the repo, examples, issues, and discussions.
