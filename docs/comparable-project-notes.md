# Comparable Project Notes

These notes summarize public positioning patterns from adjacent open-source AI agent and coding-workflow projects. They are used to improve gstack-core's public docs without copying project-specific wording.

## Projects Reviewed

- [Athena](https://github.com/bencrooks-dev/Athena): AI coding-agent workflow orchestrator with routing, verification gates, state tracking, recovery, and multi-platform install notes.
- [Agent Feed](https://github.com/fqmyysjjd/agent-feed): repository-owned workflow governance for Codex, Claude Code, Cursor, review, verification, and handoff.
- [AutoGPT](https://github.com/Significant-Gravitas/AutoGPT): agent platform with self-hosting, cloud waitlist, license boundary, examples, community, and contribution links.
- [CrewAI](https://github.com/crewAIInc/crewAI): open-source agent framework with docs links, cloud/enterprise positioning, examples, telemetry disclosure, and contribution guidance.
- [LangGraph](https://github.com/langchain-ai/langgraph): orchestration framework with concise positioning, ecosystem links, docs-first onboarding, and commercial product adjacency.

## Useful Patterns

- Lead with a clear one-line promise, then show concrete pain points solved.
- Put installation and validation commands near the top.
- Show natural command examples or real usage scenarios, not only architecture.
- Keep sustainability direction light in the README; move details into docs.
- Separate open-source core value from future cloud, enterprise, support, or maintained-pack layers.
- Add explicit trust, telemetry, privacy, and external-tool disclosure where relevant.
- Use examples, route traces, and verification artifacts as credibility signals.
- Invite contributions through capability cards, eval prompts, docs, examples, and checks.

## Positioning Implications For gstack-core

- Keep "Codex-first" as compatibility positioning, not official endorsement.
- Emphasize workflow routing, evidence gathering, clarification gates, recovery, and verified closeout.
- Make the reference-site workflow legally safer by saying "inspired, differentiated implementation" instead of "clone".
- Present future business work as a sustainability path, not as a sales pitch.
- Add public docs for brand use, governance, market notes, maintained-pack ideas, and release checks.
