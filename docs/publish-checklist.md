# Publish Checklist

Run this before creating or updating the public GitHub repository.

## Local

- `npm test` passes.
- `scripts/install-local.sh --dry-run` behaves as expected.
- `skills/gstack/SKILL.md` is concise.
- Reference files are one level deep.
- Capability cards pass validation.
- No private project names or secrets in core files.

## Open Source

- README explains what the project does.
- License exists.
- Contributing guide exists.
- Capability card spec exists.
- Evals demonstrate expected behavior.
- GitHub repository description is concise.
- Commercial roadmap is present but does not overpromise.
- Security notes explain tool-routing risks.

## Suggested Repository Description

```text
Codex-first intelligent skill router for development, creative work, GitHub, docs, recovery, and verification.
```

## Launch Notes

Position the first release as an early, local-first Codex skill router. Avoid claiming production enterprise readiness until policy, analytics, and registry features exist. Invite users to contribute capability cards and eval prompts.
