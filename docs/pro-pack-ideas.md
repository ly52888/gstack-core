# Maintained Pack Ideas

These ideas are placeholders for future maintained workflow packaging. They should not block the open-source core.

## Reference-Site Build Pack

For users who say: "Here is a reference website; help me design a differentiated implementation."

Includes:

- Browser evidence intake.
- Reference-site analysis.
- Multi-round external AI council.
- Differentiation and do-not-copy checklist.
- Information architecture draft.
- Implementation spec handoff.
- Visual QA checklist.

## Developer Pack

Includes:

- Natural command routing for bugs, screenshots, failed tasks, and current page issues.
- Codegraph-first orientation.
- Focused test selection.
- Review-before-closeout.
- Temporary artifact cleanup.
- Handoff update templates.

## GitHub Pack

Includes:

- PR review routing.
- CI failure triage.
- Issue-to-plan workflow.
- Release prep.
- Review comment addressing.
- Remote-safe publish checklist.

## Creative Pack

Includes:

- Image/video/design clarification gates.
- Brand/style identity gates.
- Tool routing between imagegen, Canva, HyperFrames, Remotion, HeyGen, and stock assets.
- Generated asset verification.

## Enterprise Policy Pack

Includes:

- Policy templates.
- Allowlist/denylist examples.
- External AI disclosure rules.
- Route trace review templates.
- Security-sensitive route escalation.
