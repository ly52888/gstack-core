# Security Notes

gstack-core routes tools and skills. Treat routing data as executable influence over an agent.

## Risks

- A malicious capability card can route work to an unsafe tool.
- A vague route can cause unwanted network or account actions.
- Creative tasks can accidentally generate misleading or off-brand assets.
- Development tasks can touch production, secrets, auth, payments, or data loss surfaces.

## Mitigations

- Keep capability cards reviewable and short.
- Use allowlists for team or enterprise setups.
- Ask before permission-sensitive GitHub, browser, cloud, payment, auth, or production actions.
- Always record route trace for non-trivial work.
- Prefer local deterministic inspection before remote actions.
- Never hide missing tools or failed verification.
