## Problem Section

```text
AI agents are powerful, but real projects fail in the gaps: unclear requirements, wrong tools, missing browser evidence, repeated context, shallow reviews, and "done" claims without verification.
```

```text
gstack-core turns those gaps into a workflow: classify the task, gather evidence, ask only what is missing, choose the smallest useful tool chain, and close with proof.
```

## Feature Bullets

- Natural command routing for bugs, failed tasks, screenshots, QA, docs, and closeout.
- Multi-round external AI council before building systems or reference-site projects.
- Capability cards for token-efficient skill/plugin/tool selection.
- Clarification gates that ask before generating low-quality assets or risky changes.
- Route trace and closeout standards for auditability.
- Bundled local codegraph for symbol search and impact analysis.
- Doctor checks for local readiness.

## Demo Scenarios

### Reference Website

```text
$gstack-skills https://example.com 我需要开发一个这样的网站，开始启动
```

Expected story:

- Browser evidence intake.
- Do-not-copy list.
- Multi-AI council.
- Differentiated MVP.
- User approval before development.

### Page Bug

```text
$gstack-skills 这个页面的任务执行失败了，帮我定位并修复
```

Expected story:

- Use current page/screenshot as evidence.
- Reproduce or inspect failure.
- Root-cause before editing.
- Verify with browser/API/tests.

### Closeout

```text
$gstack-skills 这个功能完成了，帮我收尾、清理临时垃圾并更新文档
```

Expected story:

- Run verification.
- Remove temporary files.
- Update docs/handoff.
- Report changed paths and remaining risk.
